# Agents workflow: AI и операционные агенты для платформы сайтов-приглашений

Этот документ описывает, какие AI/операционные агенты нужны платформе, какие задачи они выполняют, где они помогают владельцу, какие данные используют и что можно автоматизировать по этапам.

Главная идея: агенты не заменяют владельца бизнеса, а помогают ему быстрее обрабатывать заказы, проверять данные, создавать сайты, вести Instagram/WhatsApp и контролировать качество.

---

## 1. Зачем нужны агенты

Платформа будет расти:

- больше шаблонов;
- больше клиентов;
- больше заявок из Instagram;
- больше правок;
- больше сайтов;
- больше RSVP ответов;
- больше задач по контенту и продажам.

Если всё делать вручную, владелец быстро устанет. Поэтому система должна постепенно добавлять помощников-агентов.

Агент — это не обязательно отдельный сложный робот. На MVP этапе это может быть:

- чеклист;
- скрипт;
- подсказка в админке;
- автоматическая проверка данных;
- AI prompt;
- задача, которую запускает владелец.

В полной версии агент может работать автоматически.

---

## 2. Общая карта агентов

```text
Instagram/WhatsApp
  → Sales/Instagram Agent
  → Order Intake Agent
  → Brief Check Agent
  → Template Advisor Agent
  → Content/Text Agent
  → Build/Generator Agent
  → QA Agent
  → Publish Agent
  → RSVP Agent
  → Follow-up/CRM Agent
  → Analytics Agent
```

Основные агенты:

| Агент | Главная задача |
|---|---|
| Owner/Admin Agent | показывает владельцу, что делать сейчас |
| Sales/Instagram Agent | помогает продавать через Instagram |
| Order Intake Agent | принимает заявку и превращает её в заказ |
| Brief Check Agent | проверяет, хватает ли данных |
| Template Advisor Agent | советует подходящий шаблон |
| Content/Text Agent | пишет тексты приглашения и описания |
| Asset Agent | проверяет фото, музыку, файлы |
| Build/Generator Agent | помогает собрать клиентский сайт |
| QA Agent | проверяет сайт перед отправкой клиенту |
| Publish Agent | помогает опубликовать финальную ссылку |
| RSVP Agent | собирает и показывает ответы гостей |
| Follow-up/CRM Agent | напоминает о клиентах, оплате, отзывах |
| Analytics Agent | показывает, что работает в бизнесе |

## 3. Owner/Admin Agent

### Назначение

Главный агент для владельца. Он отвечает на вопрос:

```text
Что мне сейчас нужно сделать?
```

### Что показывает

- новые заявки;
- заказы без ответа;
- заказы с неполным брифом;
- заказы, где нужен preview;
- заказы, где клиент ждёт правки;
- заказы, где нужна оплата;
- опубликованные сайты;
- просроченные задачи.

### Какие данные использует

- `orders`;
- `clients`;
- `events`;
- `publications`;
- `payments`;
- `messages`;
- `audit_logs`.

### MVP версия

На старте это может быть простой список задач в админке:

```text
[ ] Ответить Айдане в WhatsApp
[ ] Проверить бриф Асана и Айсулуу
[ ] Отправить preview клиенту
[ ] Проверить оплату по заказу #12
```

### Полная версия

Автоматический dashboard:

- приоритеты;
- дедлайны;
- напоминания;
- быстрые кнопки;
- AI summary по каждому заказу.

## 4. Sales/Instagram Agent

### Назначение

Помогает привлекать клиентов через Instagram.

### Что делает

- предлагает идеи постов;
- пишет captions;
- пишет тексты для stories;
- готовит Reels идеи;
- предлагает ответы клиентам в Direct;
- помогает выбрать, какие шаблоны продвигать;
- готовит контент по кейсам.

### Пример задач

```text
Сделай 5 идей Reels для wedding invitation сайтов.
```

```text
Напиши короткий ответ клиенту, который спрашивает цену.
```

```text
Сделай продающий caption для premium nikah invitation template.
```

### Какие данные использует

- список шаблонов;
- пакеты и цены;
- успешные кейсы;
- типы мероприятий;
- часто задаваемые вопросы;
- стиль бренда.

### MVP версия

Владелец вручную просит агента подготовить:

- caption;
- story text;
- Direct ответ;
- список идей.

### Полная версия

Агент может вести контент-план:

- календарь публикаций;
- темы на неделю;
- подбор шаблонов для продвижения;
- напоминания, что публиковать.

---

## 5. Order Intake Agent

### Назначение

Превращает входящее сообщение клиента в понятный заказ.

### Что делает

- определяет тип мероприятия;
- вытаскивает дату из сообщения;
- определяет язык клиента;
- предлагает следующий вопрос;
- создаёт черновик заказа;
- предлагает подходящий пакет.

### Пример

Клиент написал:

```text
Здравствуйте, нужен сайт на кыз узатуу 15 августа, сколько стоит?
```

Агент должен понять:

```text
Тип: кыз узатуу
Дата: 15 августа
Интерес: цена
Следующий шаг: отправить пакеты и каталог
```

### Какие данные использует

- `clients`;
- `orders`;
- `events`;
- `packages`;
- `templates`;
- `messages`.

### MVP версия

Владелец вручную копирует сообщение и просит агента:

```text
Разбери сообщение клиента и скажи, что спросить дальше.
```

### Полная версия

Интеграция с Instagram/WhatsApp API или полуавтоматический inbox.

---

## 6. Brief Check Agent

### Назначение

Проверяет, можно ли уже делать сайт.

### Что проверяет

- есть ли имена;
- указана ли дата;
- указано ли время;
- есть ли место;
- есть ли адрес;
- открывается ли карта;
- выбран ли шаблон;
- есть ли фото;
- есть ли текст;
- нужен ли RSVP;
- есть ли ошибки или противоречия.

### Результат проверки

```text
Готово к сборке: нет
Не хватает:
- ссылка на карту
- музыка
- фото пары
- подтверждение времени
```

или:

```text
Готово к сборке: да
Можно переводить заказ в ready_to_build.
```

### Какие данные использует

- `orders`;
- `events`;
- `clients`;
- `assets`;
- `templates`.

### MVP версия

Простой чеклист в документе/админке.

### Полная версия

Автоматическая проверка формы заказа и предупреждения владельцу.

---

## 7. Template Advisor Agent

### Назначение

Помогает выбрать лучший шаблон под клиента.

### Что учитывает

- тип мероприятия;
- стиль клиента;
- язык;
- фото;
- бюджет;
- пакет;
- сезон;
- религиозный/культурный формат;
- premium или простой вариант.

### Пример

```text
Мероприятие: nikah
Стиль: luxury, исламский, тёмный фон, золото
Бюджет: premium

Рекомендация:
1. wedding-luxury-01
2. nikah-elegant-01
```

### Какие данные использует

- `templates`;
- `template_categories`;
- `packages`;
- `events`;
- `orders`.

### MVP версия

Ручная рекомендация: владелец выбирает 2–5 шаблонов.

### Полная версия

Кнопка в админке:

```text
Подобрать шаблоны
```

---

## 8. Content/Text Agent

### Назначение

Помогает писать тексты для сайта-приглашения.

### Что пишет

- hero текст;
- invitation текст;
- программа дня;
- RSVP текст;
- dress code объяснение;
- WhatsApp сообщение;
- короткие тексты для кнопок;
- тексты на русском/кыргызском/английском.

### Пример

Входные данные:

```text
Имена: Асан и Айсулуу
Событие: свадьба
Стиль: нежный, уважительный, elegant
Язык: русский
```

Результат:

```text
С радостью приглашаем вас разделить с нами особенный день — день нашей свадьбы.
Ваше присутствие станет для нас большой честью и радостью.
```

### Какие данные использует

- `events`;
- `clients`;
- `templates`;
- brand tone;
- language settings.

### MVP версия

AI помогает написать текст вручную.

### Полная версия

Тексты генерируются внутри формы/админки, но владелец подтверждает перед публикацией.

---

## 9. Asset Agent

### Назначение

Проверяет файлы клиента.

### Что проверяет

- размер фото;
- формат фото;
- качество фото;
- подходят ли изображения для hero;
- есть ли слишком тёмные/размытые фото;
- формат музыки;
- авторские/лицензионные риски;
- названия файлов.

### Результат

```text
Фото подходит для hero: да
Нужно сжать: да
Рекомендуемый crop: portrait center
```

### Какие данные использует

- `assets`;
- Supabase Storage;
- event/template settings.

### MVP версия

Владелец вручную проверяет фото.

### Полная версия

Автоматическая оптимизация:

- resize;
- webp;
- thumbnails;
- mobile versions.

---

## 10. Build/Generator Agent

### Назначение

Помогает собрать сайт клиента из данных.

### Что делает

- берёт выбранный шаблон;
- берёт event data;
- берёт assets;
- создаёт client page;
- проверяет slug;
- готовит preview build;
- сообщает ошибки сборки.

### Вход

```text
template_slug: wedding-luxury-01
event_slug: asan-aisuluu
assets: photos/music/map
settings: language, colors, RSVP
```

### Выход

```text
/clients/asan-aisuluu/
```

или preview ссылка.

### Какие данные использует

- `orders`;
- `events`;
- `templates`;
- `assets`;
- `publications`.

### MVP версия

Генерация может быть через Astro data files.

### Полная версия

Генератор берёт данные из Supabase и автоматически создаёт страницу.

---

## 11. QA Agent

### Назначение

Проверяет сайт перед отправкой клиенту и перед публикацией.

### Что проверяет

- build проходит;
- страница открывается;
- нет ошибок в console;
- фото загружаются;
- музыка работает;
- карта открывается;
- RSVP форма работает;
- кнопки работают;
- mobile view нормальный;
- нет placeholder text;
- имена/дата/время правильные.

### Checklist

```text
[ ] Build passed
[ ] Page opens
[ ] Console errors = 0
[ ] Mobile layout ok
[ ] Images load
[ ] RSVP works
[ ] Map opens
[ ] No placeholder text
```

### Какие данные использует

- build output;
- browser check;
- `publications`;
- `events`;
- `rsvp_responses`.

### MVP версия

Владелец или ассистент запускает проверку вручную.

### Полная версия

Автоматический pre-publish QA.

---

## 12. Publish Agent

### Назначение

Помогает безопасно опубликовать сайт.

### Что делает

- проверяет, что заказ готов;
- проверяет оплату;
- проверяет QA;
- создаёт/обновляет publication record;
- публикует сайт;
- возвращает финальную ссылку;
- меняет статус на `published`.

### Правила

Нельзя публиковать, если:

- не хватает данных;
- клиент не подтвердил preview;
- оплата обязательна, но не получена;
- QA не прошёл;
- ссылка содержит тестовый slug.

### Какие данные использует

- `orders`;
- `payments`;
- `publications`;
- QA result.

---

## 13. RSVP Agent

### Назначение

Помогает собирать ответы гостей.

### Что делает

- принимает RSVP responses;
- показывает список гостей;
- считает, сколько придёт;
- показывает, кто отказался;
- экспортирует таблицу;
- отправляет summary клиенту.

### Пример summary

```text
Всего ответов: 45
Придут: 38
Не придут: 7
Гостей всего: 62
```

### Какие данные использует

- `guests`;
- `rsvp_responses`;
- `events`;
- `publications`.

### MVP версия

Простая форма RSVP + таблица ответов.

### Полная версия

Dashboard для клиента/владельца + export CSV.

---

## 14. Follow-up/CRM Agent

### Назначение

Помогает не терять клиентов.

### Что делает

- напоминает ответить клиенту;
- напоминает попросить оплату;
- напоминает отправить preview;
- напоминает попросить отзыв;
- предлагает повторную продажу;
- предлагает сделать кейс для Instagram.

### Примеры напоминаний

```text
Клиент Айдана ждёт ответ 6 часов.
```

```text
Заказ #18 опубликован. Через 1 день попросить отзыв.
```

```text
Мероприятие прошло. Можно попросить разрешение показать сайт как кейс.
```

### Какие данные использует

- `clients`;
- `orders`;
- `messages`;
- `payments`;
- `audit_logs`.

---

## 15. Analytics Agent

### Назначение

Показывает, что происходит в бизнесе.

### Что показывает

- сколько заявок пришло;
- какие шаблоны популярны;
- какие типы мероприятий чаще заказывают;
- сколько заказов опубликовано;
- сколько заказов отменено;
- средний чек;
- конверсия из заявки в оплату;
- какие Instagram посты дают заявки.

### Какие данные использует

- `orders`;
- `templates`;
- `payments`;
- `clients`;
- `analytics_events`;
- `messages`.

### MVP версия

Простая таблица/ручной отчёт раз в неделю.

### Полная версия

Dashboard с графиками.

---

## 16. Минимальный набор агентов для MVP

На первом этапе не нужно строить всё сразу.

Достаточно 5 помощников:

```text
1. Owner/Admin checklist
2. Brief Check Agent
3. Content/Text Agent
4. Build/Generator Agent
5. QA Agent
```

Почему именно они:

- помогают не терять заказы;
- ускоряют создание сайта;
- уменьшают ошибки;
- дают понятный процесс;
- не требуют сложной интеграции с Instagram API.

---

## 17. Что можно отложить

Можно не делать сразу:

- автоматический Instagram inbox;
- WhatsApp API;
- сложную CRM;
- полную аналитику;
- автоматическую оплату;
- клиентский кабинет;
- авто-публикацию без проверки владельца.

Это лучше добавить после того, как первые заказы пройдут вручную.

## 19. Как агенты связаны с owner workflow

Owner workflow показывает путь заказа.

Agents workflow показывает, кто помогает на каждом этапе.

```text
Новая заявка
  → Sales/Instagram Agent
  → Order Intake Agent

Бриф
  → Brief Check Agent
  → Content/Text Agent
  → Template Advisor Agent

Создание сайта
  → Asset Agent
  → Build/Generator Agent

Проверка
  → QA Agent

Публикация
  → Publish Agent

После публикации
  → RSVP Agent
  → Follow-up/CRM Agent
  → Analytics Agent
```

## 20. Безопасность и контроль владельца

Важное правило: агент не должен делать опасные действия без подтверждения владельца.

Нужно подтверждение перед:

- публикацией сайта;
- отправкой сообщения клиенту;
- изменением цены;
- удалением заказа;
- удалением файлов;
- финальным подтверждением оплаты;
- публикацией Instagram поста;
- изменением публичной ссылки.

Агент может готовить и предлагать, но владелец подтверждает.

## 21. MVP порядок внедрения

Правильный порядок:

```text
[1] Owner/Admin checklist в документах
[2] Brief checklist в форме заказа
[3] Content prompts для текстов приглашения
[4] Build/generator script для client pages
[5] QA checklist перед preview/publish
[6] RSVP summary
[7] CRM reminders
[8] Instagram content assistant
```

## 22. Что строить дальше в коде

После этого документа следующий технический шаг — начать переводить документы в код.

Первым делом нужно сделать data-driven структуру шаблонов:

```text
astro-site/src/data/templates.ts
```

Там должны быть:

- slug;
- title;
- event type;
- package level;
- preview image;
- demo link;
- tags;
- features;
- price range;
- recommended use.

После этого каталог и страница шаблона будут брать данные не вручную, а из одного источника.

---

## 23. Итог

Агенты нужны не для красоты, а для контроля бизнеса.

Они помогают владельцу:

- быстрее отвечать клиентам;
- не забывать задачи;
- проверять брифы;
- создавать сайты;
- находить ошибки;
- публиковать аккуратно;
- собирать RSVP;
- получать отзывы;
- вести Instagram;
- понимать, какие шаблоны продаются лучше.

Для MVP достаточно начать с простых чеклистов и ручного запуска. Потом эти же процессы можно превратить в автоматизацию.
